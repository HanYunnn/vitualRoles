## 概念

兩個 AI 影像創作頻道，共用同一套工具流程。

---

## 頻道一：黑貓 Podcast

**定位：** 以黑貓為固定 IP 的 Podcast 頻道，黑貓是節目主持角色。

**製作方式：**
- 長節目（30–60 分鐘）：不做全 AI 影片，用固定黑貓角色圖 + 聲音為主，靜態或輕量循環動畫作為背景畫面
- 短精華片段（1–3 分鐘）：Higgsfield 生成場景片段，剪輯成短影音

## 工具規劃

### 核心工具：Google Flow (搭配 Google AI Ultra 方案)

**選擇原因：**
- **頂級模型整合**：結合 Veo (高畫質動態) 與 Imagen，畫面真實度與物理動態極佳。
- **Scene Builder**：可無縫拼接與延伸 (Extend) 片段，極度適合 Vlog 的連續敘事。
- **Camera Controls**：內建導演級運鏡控制，完美契合 MV 魔幻場景需求。
- **資源優勢**：已訂閱 Ultra 方案，不需額外增加初期主力工具預算。

### 備用工具：Higgsfield (若需極致懶人一致性)
- 保留其 Soul ID 作為「角色一致性」打底失敗時的備案選項。

### 完整工具組合

| 步驟 | 工具 |
|------|------|
| 角色建立與一致性 | Imagen (Google Flow 內建) 或 Midjourney (`--cref`) |
| 影片生成（MV / 場景）| Google Flow (Veo 模型 + Scene Builder) |
| 固定聲線 / 配音 | ElevenLabs |
| 嘴型對應（選配）| HeyGen 或 LivePortrait |

## 生成流程

**黑貓頻道：**
```text
Midjourney / Imagen 生成固定黑貓角色
  → 靜態 / 輕微循環動畫作為長節目背景
  → 短片段用 Google Flow (Veo) 生成動作與場景
  → ElevenLabs 配音
  → CapCut 剪輯
```

**虛擬角色頻道：**
```text
Google Flow 內建立角色設定與視覺參考 (Imagen)
  → Flow Camera Controls 生成 MV / 魔幻場景片段
  → Flow Scene Builder 無縫拼接 Vlog 日常
  → ElevenLabs 固定聲線配音
  → HeyGen / LivePortrait 處理嘴型對應
```

## MVP 快速測試步驟 (Google Flow)

1. **建立角色視覺 DNA (Imagen)**：輸入提示詞生成黑貓或虛擬人的理想靜態圖，保留 Prompt 與圖片作為日後生成的參考基準 (Reference)。
2. **測試導演運鏡 (Veo + Camera Controls)**：匯入參考圖，加入動作提示詞（如：走路、轉頭），並測試拉近 (Dolly in) 或平移 (Pan)，確認光影與物理動態是否達到 MV 品質。
3. **測試 Vlog 拼接 (Scene Builder)**：將生成的短片段透過 Scene Builder 進行延伸 (Extend) 或無縫拼接，驗證是否能流暢敘事且不跑位。
